# AI 영어 학습 웹 시스템 요구사항 v0.2

기존 요구사항에 다음 세 가지를 핵심 기능으로 추가합니다.

1. **반응형 웹·PWA 기반 학습 시스템**
2. **AI 연상 그림과 유사 단어를 활용한 플립 카드**
3. **오답에 따라 다음 문제가 즉시 달라지는 적응형 읽기 학습**

여기서 “모든 생성은 AI가 담당한다”는 것은 **사용자에게 제공되는 학습 콘텐츠를 AI가 생성한다**는 의미로 정의합니다.

AI 생성 대상은 단어, 예문, 연상법, 그림, 문장, 읽기 지문, 문제, 오답 선택지, 회화 상황, AI 상대방의 발화, 작문 과제와 교정입니다.

반면 다음 기능은 안정성을 위해 프로그램 규칙으로 관리합니다.

* 매일 제공되는 학습량
* 학습 순서와 완료 조건
* 문제 수와 회화 턴 수
* 진급 기준
* 중복 방지
* 오답 이력
* AI 결과의 데이터 형식 검증
* 개인정보와 접근 권한

즉, **AI는 콘텐츠를 만들고, 학습 엔진은 콘텐츠의 분량과 흐름을 통제합니다.**

# 1. 웹 시스템 기본 요구사항

## 1.1 서비스 형태

시스템은 데스크톱, 태블릿, 스마트폰 브라우저에서 사용할 수 있는 반응형 웹 애플리케이션으로 개발합니다.

PWA 형태를 적용하여 다음 기능을 지원합니다.

* 홈 화면에 앱처럼 설치
* 로그인 상태 유지
* 이미 생성된 학습 자료 캐시
* 일시적인 네트워크 장애 시 진행 상태 보존
* 알림 기능
* 모바일 전체 화면 학습

웹 앱 매니페스트는 웹앱 설치에 필요한 정보를 제공하며, 서비스 워커를 사용하면 앱 화면과 이미 생성된 자료를 캐시할 수 있습니다. 다만 실시간 AI 대화와 새로운 콘텐츠 생성은 네트워크 연결을 기본 전제로 합니다. ([MDN Web Docs][1])

## 1.2 음성 기능

회화와 문장 암기에서는 다음 기능을 제공합니다.

* 학습자 음성 녹음
* 음성의 텍스트 변환
* AI 문장 음성 재생
* 재생 속도 조절
* 자신의 발음 다시 듣기
* 원어민 음성과 자신의 음성 비교

브라우저 자체 음성 인식은 일부 주요 브라우저에서 지원 범위가 제한되므로, 기본 음성 인식은 서버 기반 STT를 사용하고 브라우저 음성 인식은 보조 기능으로 사용하는 것이 적절합니다. 음성 합성은 브라우저 기능을 예비 수단으로 사용할 수 있습니다. ([MDN Web Docs][2])

## 1.3 접근성

* 모든 AI 생성 이미지에 대체 텍스트를 제공해야 합니다.
* 카드의 핵심 단어나 문장을 이미지 안에 직접 넣지 않고 HTML 텍스트로 표시해야 합니다.
* 키보드만으로 플립 카드와 읽기 문제를 조작할 수 있어야 합니다.
* 드래그 방식 문제에는 버튼 또는 키보드 대체 방식을 제공해야 합니다.
* 음성을 사용하지 않아도 모든 학습을 완료할 수 있어야 합니다.
* 색상만으로 정답과 오답을 구분하지 않아야 합니다.

AI 연상 그림 역시 학습 정보이므로 대체 텍스트가 필요하며, 비텍스트 콘텐츠에 텍스트 대안을 제공해야 한다는 WCAG 원칙을 적용합니다. ([W3C][3])

# 3. 매일 학습 흐름

```text
로그인
→ 오늘의 학습 패키지 확인
→ 단어 플립 카드
→ 문장 플립 카드
→ 랜덤 AI 회화
→ 적응형 읽기
→ AI 작문 과제
```

한 번 확정된 후에는 다음 행동을 해도 내용이 바뀌지 않아야 합니다.

* 페이지 새로고침
* 브라우저 종료 후 재접속
* 다른 기기에서 로그인
* 일시적인 인터넷 연결 해제

회화 중 AI의 후속 발화와 읽기의 적응형 후속 문제만 실시간으로 생성합니다.

# 4. AI 콘텐츠 생성 시스템

## 4.1 AI 생성 범위

| 영역     | AI 생성 항목                  |
| ------ | ------------------------- |
| 어휘     | 단어 선정, 정의, 예문, 결합어, 유사 단어 |
| 연상 학습  | 연상법, 연상 장면, 이미지 프롬프트, 이미지 |
| 문장     | 암기 문장, 상황 설명, 문장 분절       |
| 회화     | 상황, 역할, 목표, 장애물, 돌발 변수    |
| 회화 진행  | AI 상대방의 매 턴 발화            |
| 읽기     | 지문, 문제, 정답, 오답 선택지, 해설    |
| 적응형 읽기 | 오답 원인을 겨냥한 새로운 후속 문제      |
| 작문     | 과제, 조건, 평가, 교정, 재작성 지침    |
| 평가     | 문법, 자연스러움, 논리, 과업 달성 평가   |

## 4.2 생성 처리 단계

```text
월별 커리큘럼
+ 학습자 레벨
+ 최근 학습 기록
+ 반복 오류
+ 최근 토픽 기록
          ↓
Daily AI Planner
          ↓
영역별 AI 콘텐츠 생성
          ↓
AI 검증 모델
          ↓
프로그램 규칙 검증
          ↓
오늘의 학습 패키지 확정
```

## 4.3 생성 모델과 검증 모델 분리

하나의 AI가 콘텐츠를 만든 뒤 스스로 정답이라고 판단하게 해서는 안 됩니다.

다음 역할을 분리합니다.

### 생성 AI

* 단어 카드 생성
* 연상 이미지 생성
* 읽기 지문과 문제 생성
* 회화 상황 생성
* 작문 과제 생성

### 검증 AI

* 단어 뜻과 예문의 일치 여부
* 읽기 문제가 지문만으로 해결 가능한지
* 객관식 정답이 하나인지
* 오답 선택지가 명백히 틀리면서도 그럴듯한지
* 회화 상황이 학습자 수준에 맞는지
* 이미지가 연상 목적에 부합하는지

### 평가 AI

* 학습자의 회화와 작문 평가
* 읽기 주관식 답안 평가
* 오류 유형 분류
* 다음 복습 항목 결정에 필요한 정보 생성

## 4.4 AI 콘텐츠 버전 관리

모든 AI 생성 자료에는 다음 정보를 저장해야 합니다.

```text
artifact_id
user_id
study_day
curriculum_version
difficulty_level
generation_seed
model_version
prompt_version
generated_at
validation_status
validation_score
safety_status
```

사용자가 학습을 시작한 후에는 같은 `artifact_id`를 계속 사용해야 합니다.

# 5. 단어 플립 카드 요구사항

## 5.1 카드 종류

### A. 연상 그림 카드

앞면:

* AI 생성 연상 그림
* 발음 재생 버튼
* 필요하면 짧은 상황 단서

뒷면:

* 영어 단어
* 품사
* 한국어 의미
* 영어 정의
* AI 연상 설명
* 예문
* 주요 결합어

예시:

```text
앞면:
폭풍이 몰아치는 동안 커다란 우산이 피해를 줄이는 그림

뒷면:
mitigate
to reduce the harmful effect of something
완화하다, 줄이다

연상:
The umbrella mitigates the impact of the storm.
```

### B. 비슷한 단어 비교 카드

AI는 비슷한 단어를 다음 유형으로 구분해야 합니다.

| 유형         | 예시                              |
| ---------- | ------------------------------- |
| 철자가 비슷한 단어 | access / assess                 |
| 발음이 비슷한 단어 | complement / compliment         |
| 의미가 비슷한 단어 | mitigate / alleviate            |
| 자주 혼동하는 단어 | economic / economical           |
| 품사가 다른 관련어 | analysis / analyze / analytical |

카드에는 단순히 비슷한 단어 목록을 제공하지 않고 다음 차이를 설명해야 합니다.

* 뜻의 차이
* 문법적 차이
* 사용 환경
* 함께 쓰는 단어
* 잘못 사용한 예문
* 올바른 예문

### C. 빈칸 카드

앞면:

> The new safety measure was intended to ______ the risk of accidents.

뒷면:

> mitigate

### D. 역방향 회상 카드

앞면:

> 위험이나 영향을 완화하다

뒷면:

> mitigate

### E. 상황 선택 카드

앞면:

> Which word best describes reducing the negative effects of a problem?

선택지:

* intensify
* mitigate
* disregard
* initiate

### F. 음성 카드

앞면에서는 음성만 들려주고 학습자가 다음을 입력하거나 말합니다.

* 들은 단어
* 단어의 뜻
* 해당 단어가 포함된 문장

# 6. 문장 암기 플립 카드

## 6.1 문장 카드 구성

앞면:

* AI 생성 상황 그림
* 한국어 상황 설명 또는 영어 의사소통 목적
* 문장을 직접 말하거나 입력하는 영역

뒷면:

* 목표 문장
* 자연스러운 음성
* 의미 단위별 문장 분절
* 한국어 의미
* 핵심 문법
* 대체 표현
* 상황에 따른 격식 수준

예시:

```text
상황:
고객센터 직원에게 부당한 요금을 설명하는 장면

목표 기능:
정중하지만 명확하게 책임을 부인하기

목표 문장:
I cannot reasonably be held liable for damage that occurred after I returned the bicycle.

문장 분절:
I cannot reasonably be held liable
/ for damage
/ that occurred
/ after I returned the bicycle.
```

## 6.2 문장 암기 모드

한 문장은 다음 방식 중 최소 두 가지로 복습합니다.

1. 그림을 보고 전체 문장 말하기
2. 한국어 상황을 보고 영어 문장 작성하기
3. 음성을 듣고 따라 말하기
4. 빈칸 채우기
5. 섞인 어절 순서 맞추기
6. 같은 의미의 다른 표현 만들기

| ID     | 요구사항                                     |

| FC-014 | 이미지 생성 프롬프트와 모델 버전을 저장해야 한다.             |

# 7. 플립 카드 복습 규칙

모든 사용자의 일일 카드 수를 같게 유지하기 위해 당일 오답 카드를 계속 반복하지 않습니다.

매일 카드 구성을 다음처럼 고정합니다.

```text
신규 단어 12개
+ 신규 문장 4개
+ 복습 카드 8개
= 총 24개
```

복습 카드 8개는 다음 우선순위로 선택합니다.

1. 전날 틀린 카드
2. 반복해서 틀린 카드
3. 장기 기억 확인 시점이 된 카드
4. 오늘의 회화에 필요한 카드
5. 학습자가 “어려움”으로 표시한 카드

당일에 복습 슬롯을 초과한 오답은 다음 날 복습 대기열에 저장합니다.

AI가 카드 내용을 생성하지만, 어떤 카드를 언제 복습하는지는 학습 엔진이 결정합니다.

# 8. 적응형 읽기 요구사항

## 8.1 기본 구성

매일 읽기 학습은 다음과 같이 구성합니다.

* AI 생성 지문 1개
* 지문 길이 400~450단어
* 핵심 문제 6개
* 적응형 후속 문제 6개
* 총 답변 문제 수 12개

각 핵심 문제 뒤에는 반드시 새로운 후속 문제를 생성합니다.

```text
핵심 문제 정답
→ 같은 능력을 더 깊게 확인하는 확장 문제 생성

핵심 문제 오답
→ 틀린 이유를 겨냥한 보정 문제 생성
```

따라서 학습자의 정답률과 관계없이 모든 학습자는 하루에 12개의 읽기 문제를 풉니다.

# 9. 오답 발생 시 처리

## 9.1 적응형 흐름

```text
핵심 문제 제시
      ↓
학습자 답변
      ↓
┌───────────────┬────────────────┐
│ 정답           │ 오답            │
├───────────────┼────────────────┤
│ 난이도 상승    │ 약한 힌트 제공  │
│ 확장 문제 생성 │ 보정 문제 생성  │
└───────────────┴────────────────┘
      ↓
새로운 후속 문제를 다음 화면에 제시
```

## 9.2 오답 보정 문제 생성 기준

새로운 문제는 단순히 기존 문제의 단어만 바꾼 문제가 되어서는 안 됩니다.

AI는 다음 정보를 사용하여 새로운 문제를 생성합니다.

* 원래 문제의 평가 능력
* 학습자가 선택한 오답
* 지문의 관련 근거 문장
* 학습자의 최근 오류 유형
* 현재 난이도
* 이미 사용한 문제 형식

예를 들어 추론 문제를 틀렸다면 다음 문제는 단순한 동일 문항 반복이 아니라 다음 중 하나로 생성합니다.

* 근거 문장 선택
* 두 문장을 연결해 결론 도출
* 잘못된 추론 찾기
* 가장 타당한 결론 선택
* 짧은 자유 답변으로 근거 설명

## 9.3 두 번째 문제도 틀린 경우

후속 문제도 틀리면 같은 날 세 번째 문제를 추가하지 않습니다.

대신 다음을 수행합니다.

1. 정답과 근거를 설명
2. 왜 선택한 답이 틀렸는지 설명
3. 오류 유형을 저장
4. 다음 날 읽기 복습 문제에 포함
5. 최초 정답률과 보정 후 성취도를 별도로 기록

이렇게 해야 일일 분량을 동일하게 유지할 수 있습니다.

# 12. 주관식 읽기 답안 처리

주관식 답안은 표현이 달라도 의미가 맞을 수 있기 때문에 다음 절차를 사용합니다.

```text
평가 AI 1차 판정
→ 별도 평가 AI 재판정
→ 두 결과 일치: 점수 확정
→ 두 결과 불일치: 정답으로 간주하지 않고 대체 객관식 문제 생성
```

AI 평가가 불확실한 답변을 임의로 오답 처리해서는 안 됩니다.

저장해야 하는 점수는 다음 두 가지입니다.

* **First-attempt accuracy:** 첫 문제의 정답률
* **Mastery accuracy:** 후속 문제까지 포함한 최종 이해도

후속 문제를 맞혔다고 해서 최초 오답 기록을 삭제해서는 안 됩니다.

* 같은 AI 역할은 최근 5학습일 동안 최대 1회
* 최근 시나리오와 의미적 유사도가 높으면 재생성
* 같은 돌발 변수를 연속 사용하지 않음

## 13.5 시나리오 고정

오늘의 회화 시나리오는 생성 시드로 고정합니다.

페이지를 새로고침해도 처음부터 다른 상황이 나오지 않습니다. 대화를 다시 시작하면 기본 상황은 같지만 AI 상대방의 세부 표현은 달라질 수 있습니다.

# 20. MVP 수용 기준

MVP는 다음 조건을 모두 만족해야 합니다.

1. 웹과 모바일 브라우저에서 동일한 계정으로 사용할 수 있다.
2. 매일 12개의 신규 단어 카드가 AI로 생성된다.
3. 모든 신규 단어 카드에 연상 그림과 대체 텍스트가 있다.
4. 매일 4개의 문장 암기 카드가 AI로 생성된다.
5. 비슷한 단어가 관계 유형과 함께 제시된다.
6. 매일 카드 학습량이 24개로 고정된다.
7. 읽기 지문과 모든 문제가 AI로 생성된다.
8. 매일 읽기 문제는 총 12개로 고정된다.
9. 핵심 문제를 틀리면 다음 문제는 해당 오류를 겨냥해 새로 생성된다.
10. 핵심 문제를 맞히면 다음 문제는 더 어려운 확장 문제로 생성된다.
11. 각 읽기 문제에는 정답의 근거 위치가 저장된다.
12. AI 검증을 통과하지 않은 문제는 학습자에게 노출되지 않는다.
13. 매일 하나의 새로운 랜덤 회화가 생성된다.
14. 회화에는 최소 8회의 학습자 발화와 하나의 돌발 변수가 포함된다.
15. 회화 도중 AI가 교사 역할로 이탈하지 않는다.
16. 회화 종료 후 교정과 재수행이 제공된다.
17. 페이지를 새로고침해도 당일 수업 내용이 변경되지 않는다.
18. 오답은 자동으로 다음 복습 대기열에 저장된다.
19. 모든 AI 생성 결과에 모델 및 프롬프트 버전이 기록된다.
20. 음성을 사용하지 않는 학습자도 전체 과정을 완료할 수 있다.
